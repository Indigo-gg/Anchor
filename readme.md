

以下是基于**“纯文本 + 交互组件 + 流程编排”**架构设计的 5 个具体工具（Agent）：

### 核心架构逻辑：LLM 作为“调度员”

*   **输入：** 你输入一句“烦死了，感觉事情永远做不完”。
*   **LLM 处理：**
    1.  识别情绪：焦虑（Anxiety）。
    2.  识别认知扭曲：灾难化思维（Catastrophizing）。
    3.  **输出指令（JSON）：**
        ```json
        {
          "tool": "Time_Slider",
          "params": {
            "theme": "dark_blue",
            "initial_text": "事情永远做不完"
          }
        }
        ```
*   **前端执行：** 瞬间调用本地写好的 `Time_Slider` 组件。

---

### 工具一：时间尺度拉杆 (The Perspective Slider)
**【针对场景】** 觉得当下的压力大到天要塌了，陷入局部的绝望。

*   **交互逻辑：**
    1.  屏幕底部出现一个横向滑块（Slider）。
    2.  左侧刻度是“现在”，右侧分别是“1周后”、“1年后”、“10年后”。
    3.  **LLM 的作用：** 预生成三段简短的文本。
*   **体验流程：**
    *   **滑块在“现在”：** 屏幕显示你刚才输入的烦恼：“事情做不完，要被骂了。”（背景深红，字体紧绷）。
    *   **你拖动到“1周后”：** 背景渐变缓和，文字变做：“这个项目已经交付了，虽然过程很累，但不管是好是坏，它结束了。”
    *   **你拖动到“1年后”：** 背景变蓝，文字：“可能会偶然想起那年有个很赶的项目，但具体的焦虑感已经记不清了。”
    *   **你拖动到“10年后”：** 文字：“这是你漫长职业生涯中的一个小插曲，它构建了你现在的韧性。”
*   **原理：** 强行拉长心理距离（Temporal Distancing），利用你“高自我洞察”的特点，用可视化交互帮你找回理智。

### 工具二：文字消融器 (The Textual Dissolver)
**【针对场景】** 脑子里有一个念头挥之不去（反刍），想摆脱但做不到。

*   **交互逻辑：**
    1.  你在输入框打出那个念头。
    2.  **LLM 的作用：** 它不评判，它只负责把这句话进行**ACT（接纳承诺疗法）式的改写**。
*   **体验流程：**
    *   你输入：“我觉得我很差劲。”
    *   屏幕正中央出现大大的字：“我觉得我很差劲。”
    *   LLM 将其改写为：“**我注意到**，我正在产生一个‘我觉得我很差劲’的想法。”（这是心理学经典的解离技术）。
    *   **交互动作：** 屏幕提供一种物理交互（比如用手指像擦玻璃一样擦拭屏幕，或者点击文字让它像沙子一样散开）。
    *   随着你的手指划过，文字真的碎裂成像素掉落到底部，最后消失。
    *   屏幕留白，只剩一句：“想法只是想法，不是事实。”

### 工具三：极速着陆 (The 5-4-3-2-1 Text Game)
**【针对场景】** 极度惊恐、解离，感觉不真实，需要在工位上立刻冷静。

*   **交互逻辑：** 这是一个基于文本的快速问答，强制转移注意力。
*   **LLM 的作用：** 充当引导者，根据你的回答给予极简的肯定。
*   **体验流程：**
    *   屏幕显示：“环顾你的工位。”
    *   LLM 指令：“在输入框里打出 **5** 个你能看到的东西。（不用过脑子，比如：杯子、屏幕...）”
    *   你快速打字。
    *   LLM：“好。现在打出 **4** 个你能摸到的东西的质感（凉的、粗糙的...）”
    *   ...直到 **1** 个你现在能控制的事情（比如：喝口水）。
*   **为什么有效：** 这种“着陆技术”需要调动感官，打字这个动作本身就是一种触觉反馈（Haptic），能把你拉回现实。

### 工具四：认知拆弹专家 (The Logic Card)
**【针对场景】** 遇到具体困难，情绪虽然不好，但理智尚存，需要解决问题的线索。

*   **交互逻辑：** 翻卡片形式。
*   **LLM 的作用：** 只有在这里，LLM 发挥它的分析能力。
*   **体验流程：**
    *   你告诉它：“老板突然加需求，我很抗拒。”
    *   LLM 瞬间生成三张卡片（不废话，直接给策略）：
        *   **卡片 A（情绪验证）：** “感到抗拒是正常的，这是你的边界感在报警，不代表你不专业。”
        *   **卡片 B（最小行动）：** “你现在能做的最小一步是什么？也许只是打开文档，写个标题？”
        *   **卡片 C（沟通话术）：** “如果不拒绝，能否换取资源？试着回复：‘可以加，但原本的A任务需要延后，您看优先哪个？’”
    *   你选择一张，卡片翻转，给你详细的行动指南。

### 工具五：静默呼吸光标 (The Breathing Cursor)
**【针对场景】** 工作中需要保持专注，但心跳很快，无法进行复杂的交互。

*   **交互逻辑：** 极简的视觉引导，不打断工作流。
*   **无需 LLM 实时生成，** 属于内置的基础功能。
*   **体验流程：**
    *   启动后，屏幕边缘出现一个半透明的细条（或者一个呼吸的光晕）。
    *   **视觉引导：** 光条向上生长（吸气 4秒） -> 停顿（保持 4秒） -> 向下收缩（呼气 4秒）。
    *   **触觉反馈（关键）：** 配合手机/手表的 Taptic Engine 震动。吸气时震动渐强，呼气时震动渐弱。
    *   你不需要盯着看，只要把手机放在手边，余光看到，或者手摸到震动，你的生理节奏就会不自觉地被它同频（Entrainment）。

---

### 技术可行性分析（为什么这个更好做？）

1.  **无需高性能 GPU：** 所有的动画（渐变、碎裂、滑动）都是前端 CSS/Canvas 动画，任何手机都能流畅运行。
2.  **LLM 响应快：** 不需要它生成长篇大论，只需要它输出 JSON 或简短句子（Few-shot prompting），API 延迟极低。
3.  **隐私与隐蔽：** 全程静音，界面可以是黑白灰的极简风，看起来就像在回消息或看笔记，完全不尴尬。
4.  **模块化开发：** 你可以先做“工具一”和“工具五”，验证对自己有效了，再慢慢加其他的。
