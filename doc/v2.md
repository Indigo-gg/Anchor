| 模块 | 优势 | V2 策略 |
|------|------|---------|
| **角色系统** | 前端 CRUD、可视化切换、角色隔离会话/记忆 | **保留并增强**，作为核心差异化 |

### 1.3 V1 的痛点 (需要解决)

| 痛点 | 说明 |
|------|------|
| 工具扩展性 | 新增工具需要改 3+ 个文件，无法动态加载 |
| 工具生态为零 | 所有工具自己写，无法复用社区成果 |
| 记忆系统简陋 | Jaccard 文本匹配，无语义理解 |
| 模型绑定 | 硬编码 2 个提供商，切换困难 |
| Agent 能力 | 无子 Agent、无循环检测、无上下文窗口管理 |

### 3.2 OpenClaw Skill 格式解析

OpenClaw 的 Skill 是一个 Markdown 文件 (`SKILL.md`)，结构为:

```markdown
---
name: weather
description: "Get weather forecasts for any location"
metadata:
  openclaw:
    emoji: "🌤️"
    requires:
      bins: ["curl"]
---

## When to use

Use this skill when the user asks about weather, forecasts, or climate.

## How to use

Run the following command:
```bash
curl "wttr.in/{location}?format=3"
```

## Examples

- "What's the weather in Tokyo?" → `curl "wttr.in/Tokyo?format=3"`
```

**关键点**: Skills 不是可执行的工具函数，而是 **注入到 system prompt 中的指令文本**，教 Agent 如何使用 bash/API 完成任务。

### 3.3 工具类型统一模型

```typescript
// === 新的统一工具接口 ===

/** 工具来源 */
type ToolSource = 'builtin' | 'openclaw-skill' | 'custom'

/** 工具执行模式 */
type ToolExecMode =
  | 'kernel'          // 内置交互工具 (进入 Kernel Mode 多轮对话)
  | 'instant'         // 内置即时工具 (单次执行)
  | 'prompt-inject'   // OpenClaw Skill (注入 system prompt, 由 Agent 自行调用 bash/API)

/** 统一工具定义 */
interface UnifiedTool {
  key: string
  name: string
  icon: string
  description: string
  source: ToolSource
  execMode: ToolExecMode

  // 内置工具字段 (source === 'builtin')
  builtinMeta?: ToolMeta        // 现有的 ToolMeta 完整定义

  // OpenClaw Skill 字段 (source === 'openclaw-skill')
  skillMeta?: {
    markdownBody: string        // Skill 的 Markdown 正文 (注入 prompt)
    requires?: {                // 依赖检查
      bins?: string[]
      env?: string[]
    }
    platform?: string[]         // 平台限制 ['darwin', 'linux', 'win32']
  }
}
```

### 3.4 Skill 加载器实现

```typescript
// src/services/skill-loader.ts

import fs from 'fs'
import path from 'path'
import matter from 'gray-matter'    // 需要新增依赖: gray-matter

/** 内置 Skills 目录 (从 OpenClaw 仓库复制/同步) */
const SKILLS_DIR = path.join(__dirname, '../../skills')

/** 用户自定义 Skills 目录 */
const USER_SKILLS_DIR = path.join(app.getPath('userData'), 'skills')

interface SkillManifest {
  name: string
  description: string
  metadata?: {
    openclaw?: {
      emoji?: string
      os?: string[]
      requires?: {
        bins?: string[]
        env?: string[]
      }
    }
  }
}

/** 加载单个 Skill */
function loadSkill(skillDir: string): UnifiedTool | null {
  const filePath = path.join(skillDir, 'SKILL.md')
  if (!fs.existsSync(filePath)) return null

  const raw = fs.readFileSync(filePath, 'utf-8')
  const { data, content } = matter(raw)               // 解析 YAML frontmatter
  const manifest = data as SkillManifest

  // 平台检查
  const platform = manifest.metadata?.openclaw?.os
  if (platform && !platform.includes(process.platform)) return null

  return {
    key: `skill_${manifest.name}`,
    name: manifest.description || manifest.name,
    icon: manifest.metadata?.openclaw?.emoji || '🔧',
    description: manifest.description || '',
    source: 'openclaw-skill',
    execMode: 'prompt-inject',
    skillMeta: {
      markdownBody: content.trim(),
      requires: manifest.metadata?.openclaw?.requires,
      platform: manifest.metadata?.openclaw?.os,
    }
  }
}

/** 加载所有可用 Skills */
export function loadAllSkills(): UnifiedTool[] {
  const skills: UnifiedTool[] = []
  for (const dir of [SKILLS_DIR, USER_SKILLS_DIR]) {
    if (!fs.existsSync(dir)) continue
    for (const entry of fs.readdirSync(dir)) {
      const skill = loadSkill(path.join(dir, entry))
      if (skill) skills.push(skill)
    }
  }
  return skills
}
```

### 4.2 通信协议选择

OpenClaw Gateway 暴露两种接口:

| 接口 | 协议 | 特点 | 推荐场景 |
|------|------|------|---------|
| `/v1/responses` | HTTP + SSE | OpenAI 兼容 API，简单易用 | **推荐: Phase 2 初期** |
| ACP WebSocket | WS | 双向实时，支持子 Agent 事件 | Phase 2 后期，需要更多控制 |

#### `/v1/responses` API 调用示例

```typescript
// src/services/openclaw-client.ts

const GATEWAY_URL = 'http://127.0.0.1:18789'

interface OpenClawRequest {
  model: string                          // 由角色配置决定
  input: string | InputItem[]
  instructions?: string                  // 角色的 systemPrompt
  tools?: ClientToolDefinition[]         // Anchor 专属工具 (客户端执行)
  stream: boolean
  user?: string                          // 映射到 sessionKey
}

/** 发送消息到 OpenClaw Gateway (流式) */
async function sendToGateway(
  request: OpenClawRequest,
  onTextDelta: (delta: string) => void,
  onToolCall: (name: string, args: Record<string, unknown>) => void,
  onComplete: () => void
) {
  const response = await fetch(`${GATEWAY_URL}/v1/responses`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${gatewayToken}`,
    },
    body: JSON.stringify({ ...request, stream: true }),
  })

  const reader = response.body!.getReader()
  const decoder = new TextDecoder()

  while (true) {
    const { done, value } = await reader.read()
    if (done) break

    const chunk = decoder.decode(value)
    for (const line of chunk.split('\n')) {
      if (!line.startsWith('data: ')) continue
      if (line === 'data: [DONE]') { onComplete(); return }

      const event = JSON.parse(line.slice(6))
      switch (event.type) {
        case 'response.output_text.delta':
          onTextDelta(event.delta)
          break
        case 'response.output_item.done':
          if (event.item.type === 'function_call') {
            onToolCall(event.item.name, JSON.parse(event.item.arguments))
          }
          break
      }
    }
  }
}
```

// src/services/role-sync.ts

import type { RoleDefinition } from './role'

/** Anchor RoleDefinition → OpenClaw Agent 配置 */
function roleToAgentConfig(role: RoleDefinition) {
  return {
    identity: {
      name: role.name,
      description: role.description,
    },
    // 角色的 systemPrompt 成为 instructions
    systemPrompt: role.systemPrompt,

    // 模型由角色配置驱动 (V2 可在 RoleEditor 新增模型选择)
    model: role.model || 'gemini-2.5-pro',

    // 工具策略
    tools: {
      allow: role.toolKeys
        .filter(k => !k.startsWith('anchor_'))  // OpenClaw 标准工具
        .map(k => k.replace('skill_', '')),
      // Anchor 内置工具通过 client tools 注册，不在此处
    },

    // Skills
    skills: {
      enabled: role.toolKeys
        .filter(k => k.startsWith('skill_'))
        .map(k => k.replace('skill_', '')),
    },

    // 记忆
    memory: role.memory ? { enabled: true } : { enabled: false },
  }
}

/** 从 Agent 配置恢复 RoleDefinition (双向同步) */
function agentConfigToRole(config: AgentConfig, roleId: string): RoleDefinition {
  return {
    id: roleId,
    name: config.identity.name,
    description: config.identity.description || '',
    icon: '',     // OpenClaw 不存储 icon
    color: '',    // OpenClaw 不存储 color
    systemPrompt: config.systemPrompt || '',
    toolKeys: [
      ...config.tools?.allow || [],
      ...(config.skills?.enabled || []).map(s => `skill_${s}`),
    ],
    useCommonTools: true,
    memory: config.memory?.enabled ?? false,
    isBuiltin: false,
  }
}

**调用流程**:

```
1. Anchor 发送请求到 Gateway，附带 client tools 定义
2. Gateway 的 Agent 判断需要调用某个 client tool (如 breathing_guide)
3. Gateway 返回 status: "incomplete" + function_call 输出
4. Anchor 拦截 function_call，在前端渲染对应的 Vue 工具组件
5. 用户完成工具交互后，Anchor 将结果作为 function_call_output 发回 Gateway
6. Gateway 继续 Agent 循环
```

    instructions: currentRole.systemPrompt,
    tools: getAnchorClientTools(),
    stream: true,
    user: `anchor:${currentRole.id}:${sessionId}`,
  },
  // 处理文本流
  (delta) => appendToStream(delta),
  // 处理工具调用 → 渲染 Vue 组件
  async (toolName, args) => {
    if (isAnchorTool(toolName)) {
      // 渲染对应的 Vue 工具组件
      const result = await renderToolComponent(toolName, args)
      // 将结果发回 Gateway
      await sendToGateway({
        input: [{
          type: 'function_call_output',
          call_id: currentCallId,
          output: JSON.stringify(result),
        }],
        // ... 其他参数
      })
    }
  },
  () => finishStream())
}
```

### 4.5 OpenClaw Gateway 生命周期管理

Anchor 需要管理 OpenClaw Gateway 的生命周期:

```typescript
// src/services/gateway-manager.ts (Electron main process)

import { spawn, ChildProcess } from 'child_process'

let gatewayProcess: ChildProcess | null = null

/** 启动 OpenClaw Gateway (随 Electron 启动) */
async function startGateway() {
  // 检查是否已有 Gateway 运行
  try {
    const res = await fetch('http://127.0.0.1:18789/health')
    if (res.ok) return   // 已经在运行
  } catch {}

  gatewayProcess = spawn('npx', ['openclaw', 'gateway', '--port', '18789'], {
    env: {
      ...process.env,
      OPENCLAW_PROFILE: 'anchor',     // 使用 Anchor 专属配置
    },
    stdio: 'pipe',
  })

  // 等待 Gateway 就绪
  await waitForGateway()
}

/** 随 Electron 退出时关闭 */
app.on('before-quit', () => {
  gatewayProcess?.kill()
})
```

## 6. 迁移兼容性方案

### 6.1 引擎切换策略 (V1 ↔ V2 共存)

Phase 2 不是一步到位替换 V1 引擎，而是 **双引擎共存，用户可选**:

```typescript
// src/services/engine.ts

type EngineMode = 'v1-local' | 'v2-openclaw'

/** 获取当前引擎模式 */
function getEngineMode(): EngineMode {
  // 检查 OpenClaw Gateway 是否可用
  // 用户可在设置中手动选择
  return localStorage.getItem('anchor_engine_mode') as EngineMode || 'v1-local'
}

/** 发送消息 (根据引擎模式路由) */
async function sendMessage(message: string, role: RoleDefinition) {
  const mode = getEngineMode()

  if (mode === 'v2-openclaw') {
    return sendToGateway(...)    // OpenClaw Gateway
  } else {
    return sendToAgent(...)      // 现有 V1 引擎
  }
}
```

### 6.2 数据迁移

| 数据 | V1 存储 | V2 存储 | 迁移方式 |
|------|---------|---------|---------|
| 角色定义 | localStorage `anchor_roles` | localStorage (保留) + OpenClaw config 同步 | 自动双写 |
| 会话历史 | localStorage `anchor_sessions` | OpenClaw JSONL 文件 | 首次启动时单次导入 |
| 记忆 | localStorage `anchor_memories` | OpenClaw sqlite-vec | 首次启动时 `openclaw memory index` |
| 设置 | localStorage 各项 | 保留在 localStorage | 不迁移 |

## 7. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| OpenClaw 项目不稳定/停更 | 引擎无法升级 | V1 引擎永久保留作为 fallback |
| Gateway 启动慢/占资源 | 用户体验下降 | 后台预启动 + 最小化配置 |
| Skills 格式变更 | 加载器失效 | 版本锁定 + 适配层隔离 |
| Client Tool 协议变更 | 工具调用失败 | 固定 API 版本 `/v1/responses` |
| 向量记忆不如预期 | 记忆质量下降 | 保留 V1 记忆作为补充 |
| 角色同步冲突 | 数据不一致 | Anchor 端为主 (single source of truth) |
